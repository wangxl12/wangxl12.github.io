python中的__init__.py,__all__,import,__name__,__doc__

发表于 2022-01-19 分类于 编程语言 阅读次数： Valine：

Summarize usage of __init__.py, __all__, import, __name__, __doc__ .

__init__.py文件

用来表明一个文件夹是python包还是一般文件夹，如果存在该文件，就是python包。用处：

  1. 当用于import 对象时，可以识别出哪些是可以导入的包，哪些是一般文件夹。

  2. 可以在其中导入需要的对象，然后通过在执行程序中import package来引用这些包，所以简化了执行程序，因为每次导入package的时候会自动执行__init__.py；但最好不要在这里写自己的模块，该文件越简单越好；

  3. 设置__all__参数，只用于指定 from package import * 时，导入的包是哪些，不需要的包可以先不导入；并不影响from package import module/package、import package.module等形式的导入。

import

import到底干了啥？从package导入模块需要使用from package import …形式。

__name__

用于判断当前模块是不是主程序文件（主执行），也就是查看 __name__的值是否为 __main__，如果是则该程序属于主程序文件，如果不是则显示该文件的文件名。所以很多时候看到有文件里有：if __name__ == "__main__":，当主程序是该文件的时候才执行这里接下来的内容，如果将其作为导入文件就不会执行。

__doc__

__doc__：模块的注释文本，例如函数或者类的说明，用 ‘’’…’’’ 三引号形式包围
