Gitee自动部署博客

发表于 2021-12-01 分类于 Git 阅读次数： Valine：

Gitee Pages访问速度快，所以打算弄一个Gitee的镜像，但是它不像GitHub Pages那样，提交代码可以实现自动更新Pages，因为Gitee的自动部署属于Gitee Pages Pro的服务。每次提交之后我们需要手动点击【更新】，甚是麻烦。

为此，有大佬写了Gitee Pages Action来解决这个问题，只要在某个仓库配置好这个动作就能在往这个仓库提交任何内容时自动让Gitee部署Gitee里面指定仓库的内容。

操作步骤

首先默认已经在本地生成了ssh key，其中id_rsa是私钥，id_rsa.pub是公钥，公钥已经放在了github和gitee上的ssh key中。

现在打开GitHub项目里的settings/Secrets，新建一个repository secret，然后将私钥内容放入value中，命名为GITEE_RSA_PRIVATE_KEY:

然后再新建一个repository secret，值为Gitee账号的密码，命名为GITEE_PASSWORD：
